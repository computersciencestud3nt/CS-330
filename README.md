Reflection – CS 330 Computational Graphics and Visualization
How did I approach designing this software?

I approached designing this software by first identifying the core objective and breaking the project into manageable components. For this 3D scene, I began by visualizing the environment I wanted to create and gathering reference images to guide my design decisions. I then built the scene step by step, starting with the core objects before adding lighting and textures. I focused on modular design and structured planning, which helped reduce complexity as the project evolved and ensured that each component fit cohesively into the overall scene.

How do I approach developing programs?

I approach development by implementing functionality incrementally and validating each component before expanding it. I prioritize clean structure, meaningful naming conventions, and a logical separation of responsibilities. In this project, I carefully implemented transformation functions, lighting models, and shader configurations while ensuring that each object rendered correctly before introducing additional complexity. This methodical approach allowed me to debug efficiently and maintain control over the system as it grew.

How can computer science help me reach my goals?

Computer science provides the foundation for problem solving, logical reasoning, and technical creativity. It allows me to build interactive systems and visual simulations that blend software with physical design. My long-term goal is to create interactive museum exhibits by combining my carpentry skills with computer engineering and programming. By integrating hardware, software, and immersive visual experiences, I hope to design educational environments that are both engaging and technologically innovative.
